# Local-Sentinel
Local Sentinel Monitor** is a local software suite for precision OSINT monitoring of target Telegram channels and chats. The project addresses the issue of automated data collection, filtering, and semi-automated publication in a noisy environment.

## Описание проекта
**Local Sentinel Monitor** — это локальный программный комплекс для прецизионного OSINT-мониторинга целевых Telegram-каналов и чатов. Проект решает задачу автоматизированного сбора, фильтрации и полуавтоматической публикации данных в условиях высокого информационного шума. 

Система построена на архитектуре «Разделенного Контура», изолирующей процесс сбора данных (User API) от процесса публикации (Bot API), что обеспечивает высокую устойчивость к блокировкам (Flood Wait / Ban). Оптимизирована для локального развертывания с мониторингом до 40 целевых источников.

## Полный спектр возможностей
Проект реализует полный цикл обработки информации (Pipeline):

1. **Пассивный мониторинг в реальном времени:** - Захват событий (сообщений) исключительно после старта системы.
   - Поддержка закрытых и открытых чатов/каналов.
2. **Интеллектуальная фильтрация (Ядро):**
   - **Blacklist Validator:** Мгновенное отсечение контента от заблокированных пользователей.
   - **Semantic Scanner:** Поиск по точному вхождению ключевых слов и регулярным выражениям.
   - **Smart Deduplication:** Хеширование сообщений для предотвращения ретрансляции дублирующего спама или ранее отклоненного текста.
3. **Асинхронная база данных:**
   - Использование SQLite в режиме WAL для предотвращения блокировок при конкурентном доступе от двух скриптов.
   - Буферизация и формирование очереди задач (Pending Status).
4. **Интерфейс модератора (Telegram Bot):**
   - Интерактивные карточки сообщений (Источник, Автор, Текст).
   - FSM (Finite State Machine) для режима `[📝 Редактировать]`.
   - Быстрые действия: `[✅ Опубликовать]`, `[🚫 Бан]`, `[📁 В архив]`.

## Структура репозитория
Архитектура разделена на независимые модули (Separation of Concerns).

```text
├── core_collector/         # Контур А: Скрипт-слушатель (Telethon)
│   ├── client.py           # Инициализация User API
│   └── handlers.py         # Логика перехвата событий
├── bot_moderator/          # Контур Б: Бот-модератор и издатель (Aiogram)
│   ├── bot.py              # Инициализация Bot API
│   ├── keyboards.py        # Inline-кнопки
│   └── states.py           # Машина состояний для редактирования
├── database/               # Слой данных
│   ├── models.py           # Схема таблиц (ORM)
│   └── queries.py          # CRUD операции и фильтрация
├── .env.example            # Шаблон конфигурации окружения
├── config.py               # Глобальные настройки и список источников
├── requirements.txt        # Зависимости проекта
└── README.md               # Документация проекта
